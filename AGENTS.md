# AGENTS.md

## Repository Rules

- 設定ファイルに修正、更新などを行った場合は必ずバージョンを上げてください。
- どのような場合でも、ユーザー対応は必ず日本語で行ってください。
- 作業は最後まで一気に仕上げてください。作業終了時には GitHub にコミットし、プッシュし、最後にビルドまで行ってください。
- 通常ビルドおよびリリースビルドを実行する前には、対象と同一カテゴリーかつ同一プロジェクトに属する実行中タスクや関連プロセスを必ず停止し、ビルド成果物や target ディレクトリのロックを解消してから実行してください。
- 通常ビルドの出力先は `target\\debug`、リリースビルドの出力先は `target\\release` を原則固定とし、`CARGO_TARGET_DIR` などによる別出力先の利用はロック回避や検証分離が必要な場合に限る例外運用として扱ってください。
- 作業はできるだけ Agent Teams を組んで行ってください。
- 通知リングシステムは最重要機能として扱い、関連不具合の検出・修正を最優先してください。
